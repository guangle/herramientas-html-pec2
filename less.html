Cuando domines Sass no querrás volver atrás. Todo el que ha vivido de cerca el desarrollo web en los últimos años sabe que CSS3 ha supuesto una verdadera revolución y que cada vez las web lucen mejor y hoy día se pueden hacer cosas increibles pero por otra parte, el nivel de exigencia es cada vez mayor y el código css de nuestros proyectos (que cada vez son más grandes) crece y crece y crece..

Sass nació para llevar al mundo de la escritura de estilos la filosofía DRY, esto es, Don't repeat yourself e intentar por todos los medios que escribamos código repetido, algo que sucecía frecuentemente cuando escribíamos CSS 'normal'. Esto lo consigue añadiendo algunas funcionalidades provinientes del mundo de la 'programación pura' como variables, funciones, imports, etc

Las principales características de Sass (por Syntatically Awesome Style Sheets) son:

  • Variables para almacenar un valor que repetiremos multiples veces en nuestros estilos
  • Haremos nuestros estilos mucho más fácil de leer con anidamiento o nesting
  • Los Mixins son algo parecido a las funciones, que pueden recibir parámetros y aplicarse en distintas partes de nuestro código
  • Podemos partir (modularizar) nuestros ficheros para hacer nuestro proyecto más mantenible
  • ¿Tus clases comparten algunas propiedades? Es posible que te venga bien usar la herencia e extends

Solo se han citado algunas de las características más importantes de la herramienta, Sass es realmente potente y poco a poco su uso se está convirtiendo en un estándar en el desarrollo web moderno. De hecho, los datos de uso reflejan que es la alternativa más usada en cuanto a procesadores css se trata, pero ¡no es en absoluto la única alternativa! Si quieres puedes echarle un ojo a Less

Un poco de código

Veremos alguna de las funcionalidades que ofrece Sass

Variables


              /* Definicion de variables */
              $color1: rgba(130, 35, 178, 1);
              $color2: rgba(217, 3, 104, 1);
              /* Uso de las variables declaradas */
              p::first-letter {
                font-size: 250%;
                color: $color1;
                font-weight: bold;
              }
Declaración y uso de variables

Anidamiento


                .contenido {
                  h1 {
                    color: $color1;
                  }
                }
El nesting nos permite que el código sea más legible, resulta 'intuitivo' ver que únicamente aplicamos el color a los h1 que estén dentro de elementos con la clase 'contenido'

Mixins


                @mixin caja($tam, $color) {
                  width: $tam;
                  height: $tam;
                  background-color: $color;
                }
                
                .caja-grande-roja {
                  @include caja(200px, rgb(255,0,0));
                  /* + atributos adicionales..*/
                }
                
                .caja-peque-azul {
                  @include caja(30px, rgb(0,0,255));
                  /* + atributos adicionales..*/
                }
Los mixins nos permite reutilizar mucho código css, construyendo 'funciones' que apliquen ciertos estilos comunes a elementos
