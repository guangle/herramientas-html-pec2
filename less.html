Less sigue exactamente la misma filosofía que Sass, se escriben los ficheros de estilos en ficheros con un formato propio, donde contaremos con funcionalides añadidas al css estándar y tras un trabajo de procesamiento, obtendremos un único fichero con formato .css que es el que serviremos y sabrá interpretar nuestro navegador.

En este caso, los ficheros que procesemos con less deberán tener la extensión .less y tendremos disponibles una serie de utilidades que nos facilitarán mucho la labor de aplcar los estilos a una web: anidamento, variables, operadores, etc. Una de las características de less que más llama la atención es que permite que la compilación se haga, mediante javascript, en el navegador del cliente, aunque obviamente esta opción penaliza el rendimiento de la web.

Las principales características de Less son:

  • Variables para almacenar un valor que repetiremos multiples veces en nuestros estilos
  • Anidamiento de propiedades, con lo que conseguimos un código más organizado y fácil de leer
  • Los Mixins son algo parecido a las funciones, que pueden recibir parámetros y aplicarse en distintas partes de nuestro código
  • Podemos usar operadores (+,-,*,/) para el cálculo de nuestros anchos, márgenes, paddings.. ¡incluso podemos operar con colores!
  • Podemos importar otros ficheros (less o css) usando la palabra reservada @import

Tanto Sass como Less ofrecen características muy similares así que el uso de una u otra herramienta depende de las preferencias de cada desarrollador porque a nivel de funcionalidad son prácticamente equivalentes. Eso sí, parece que Sass es algo más popular en el mundo del desarrollo, luego nos será mas sencillo encontrar documentación, tutoriales y videos.

Impresionante! los preprocesadores css nos permiten ahorrar gran cantidad de tiempo y llegar a diseños impresionantes con muy poco esfuerzo.

Un poco de código

Veremos alguna de las funcionalidades que ofrece Less

Variables


              /* Definicion de variables */
              @color1: rgba(130, 35, 178, 1);
              @ancho1: 10px;
Declaración y uso de variables

Anidamiento


                /* Código Css */
                #cabecera {
                    color: black;
                }
                #cabecera .navigation {
                    font-size: 12px;
                }
                #cabecera .logo {
                    width: 300px;
                }
                /* Código equivalente en less */
                #cabecera {
                    color: black;
                    .navigation {
                        font-size: 12px;
                    }
                    .logo {
                        width: 300px;
                    }
                }
En este caso, no ahorramos muchas líneas pero el código esta mejor organizado y es mucho más 'intuitivo', ya que a 'simple vista' se aprecian que navigation y logo solo existirán dentro de un elemento conn id=cabecera

Mixins


                    @padding_base: 5%;
                    @padding_1: @padding_base * 1,15;
Los operadores aritméticos nos pueden ser útiles para definir tamaños en base a una referencia que tengamos declarada en nuestra aplicación

Pon a pruebas tus conocimientos con estos test
Como la herramienta está basada en javascript, con Less podríamos no procesar los ficheros .less, sino mandarlos al cliente y que se transformen en el navegador en css
  • Falso, eso no se puede hacer
  • Verdadero, es la forma recomendable de trabajar con less
  • Verdadero, pero penalizaríamos el rendimiento de la página
¿Cómo definimos en Less una variable con nombre color y valor #abcabc?
  • %color: #abcabc;
  • @color1: #abcabc;
  • $color : #abcabc;
El nesting o anidamiento tiene como ventaja..
  • Tener el código más organizado y hacerlo más fácil de leer
  • Ofuscar el código, para que nadie pueda copiarnos
  • Reduce el tiempo de procesado del fichero .less
