Los module boundlers son herramientas que nos harán la vida mucho más fácil durante el proceso de construcción y despliegue de una aplicación web moderna, ya que consiguen automatizar una serie de tareas comunes que hay que hacer en todos los desarrollos y, sobre todo, gestionar las dependencias del proyecto. Una página web no es un único fichero ¡ni mucho menos! más bien son algunos ficheros de con el contenido html, otros donde escribimos los estilos y otros en los que escribimos el comportamiento de la aplicación. Los ficheros de cada uno de estos grupos pueden estar escrito en diferentes formatos o lenguajes. Además, afortunadamente, cuando escribiemos una web no lo hacemos todo desde cero, sino que nos basamos en librerías, herramientas de terceros, frameworks con estilos..

Todo esto tiene como resultado que un proyecto web puede tener cientos de ficheros, o modulos, que están escritos en diferentes lenguajes y que dependen unos de otros. Para gestionar todo este embrollo están los module boundlers, que 'traducirán' cada fichero a un formato que entienda el navegador y sabrá ordenadar todos los módulos de nuestro proyecto para que se resuelvan las dependencias de forma adecuada. Quiero saber más

Algunos ejemplos de los tipos de tareas con los que nos puede ayudar un module boundler.. Ejemplo1: Escribimos los estilos de nuestra aplicación en diferentes ficheros sass. Nuestro module boundler se encarga de coger todos los ficheros scss y transformarlos en un unico fichero .css (que es el que entiende el navegador), aplicando también minificación para que el resultado sea más ligero y su descarga sea más rápida. Ejemplo 2: Utilizamos typescript porque estamos acostumbrados a tener nuestras variables tipadas. Como el navegador no entiende este formato, el module boundler lo traducirá a javascript 'puro' para que pueda ser ejecutado en el navegador.

Como sabrás, una web no se escribe directamente en el entorno real o de producción, sino que durante el proceso de construcción los desarrolladores prueban sus cambios y el contenido que van escribiendo en un entorno de desarrollo. Generalmente, un module boundler realiza una serie de tareas u otras en función del entorno para el cual este trabajando. En el entorno de desarrollo el objetivo principal es hacerle fácil la vida al desarrollador: ofreciendo un entorno de desarrollo, mostrando mensajes de error fáciles de entender, aplicando los cambios en la página cuando se re-escriban ficheros, etc. Por el contrario, la premisa principal cuando se contruye una web para el entorno de producción es que la web funcione de la forma más rápida posible. Para ello, los module boundler aplican procesos de concatenación, minificación de ficheros y optimización de imágenes y recursos estáticos.
