WebPack es un module bundler, que se podría traducir como 'empaquetador de módulos'. Es decir, una herramienta que nos ayudará en la construcción de páginas web que se encarga de organizar nuestros ficheros, gestionando las dependencias que existan entre ellos y aplicando las transformaciones necesarias en estos ficheros en función de dónde se está ejecutando nuestra página.

La carga de dependencias se hace de manera recursiva y de forma transparente para el desarrollador, de forma que este se puede despreocupar de los cientos o miles de ficheros que aparecen debajo del directorio node_modules. Con sus distintos loaders, webpack permite importar y procesar distintos tipos de archivos (css para preprocesar, javascript, imágenes, html,..) y generar un output, que son los ficheros que finalmente se enviarán al navegador del cliente que use nuestra web.

Aunque el cometido de un module bundler es gestionar y empaquetar los distintas partes (módulos) que componene una aplicación, este tipo de herramientas normalmente hacen muchísimas cosas más, en el caso de WebPack es una de las herramientas más potentes por su amplia funcionalidad y versatilidad y es practicamente un estándar en la construcción del front-end en aplicaciones web:

Entre las 'otras cosas' que podemos hacer con webpack:

  • Gestiona todos los módulos de nuestra web y sus dependencias
  • Concatenación de código
  • Minificación, para que el código ocupe menos espacio y pueda viajar más rápido por la red
  • Ofuscación para, si así lo deseamos, nadie pueda entender nuestro código en el lado del cliente
  • Convertir formatos de nuestros ficheros. Por ejemplo, si nos es más útil o cómodo, podemos usar TypeScript y posteriormente WebPack se encargará de transformar esos ficheros a código javascript 'puro' (que es el que entienden los navegadores).
  • Uno de los puntos claves de webpack es que es ampliable mediante el uso de plugins

Al ofrecer tantas funcionalidades, webpack se está convirtiendo casi en un estándar en el desarrollo web, ya que practicamente no es necesaria ninguna herramienta adicional para realizar todas las tareas necesarias durante el proceso de construcción y despligue. Por ejemplo, usando otros module boundlers como parcel es normal que necesitemos una herramienta adicional para ejecutar tareas (como Gulp)

¡No todo va a ser perfecto! Quizás el principal inconveniente que tiene webpack es que puede ser un poco complicado de utilizar al principio. Se podría decir que webpack es más útil en proyectos de tamaño medio, donde los desarrolladores están experimentados. Para proyectos sencillos que no requieran las funcionalidades avanzadas que ofrece webpack quizás sea mejor usar otra alternativa como Parcel
